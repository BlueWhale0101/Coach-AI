Fitness Logging & Analysis System

A conversational fitness tracking and coaching platform built around ChatGPT, Supabase, and a lightweight web administration interface.

The system is designed around a simple philosophy:

Logging must remain conversational.

Storage must be structured.

Raw text must always be preserved.

Rather than forcing the user into forms, menus, and rigid workflows, the system allows natural conversation with a coaching GPT while storing structured data for analysis and long-term tracking.

⸻

Goals

The project has four primary objectives:

1. Preserve conversational fitness logs.
2. Extract structured fitness data.
3. Support coaching and trend analysis.
4. Minimize user logging burden.

The user should be able to say:

“Weighed 198.0. Hard HYROX today. Brekky was eggs and mushrooms.”

and continue with their day.

The system handles storage, extraction, analysis, and reporting automatically.

⸻

Core Philosophy

Raw Text is Sacred

Every log is stored exactly as entered.

Raw text is the authoritative record.

Structured records are derived from raw text and may be corrected later.

⸻

Facts vs Estimates vs Interpretation

The system deliberately separates:

Facts

Information explicitly stated by the user.

Examples:

* bodyweight
* meals consumed
* workouts performed
* low energy
* high hunger

Estimates

Information generated by GPT.

Examples:

* calorie estimates
* protein estimates
* workout duration estimates
* training load estimates

Estimates are generally stored as ranges.

Interpretation

Coaching analysis.

Examples:

* possible under-fueling
* recovery concerns
* excessive fatigue
* positive progress indicators

Interpretation is stored separately from user facts.

⸻

Null is Better than Wrong

Missing information remains missing.

The system does not invent data.

If the user says:

“I am tired.”

The system records fatigue.

The system does not automatically assume:

* poor sleep
* illness
* soreness
* stress

unless explicitly stated.

Data Model

log_entries

Stores original conversational logs.

One user message creates one log entry.

Examples:

Weighed 198.0.
Workout:
5 rounds:
20 cal row
10 hang cleans
Breakfast:
3 eggs and mushrooms.

⸻

fitness_events

Structured records extracted from log entries.

Examples:

* meal
* workout
* weigh_in
* recovery
* sleep
* symptom
* social_event
* alcohol

Each event references its originating log entry.

⸻

coach_observations

Stores coaching analysis.

Examples:

* under-fueling concerns
* recovery recommendations
* training observations
* behavioral trends

Coach observations are not factual user data.

⸻

daily_summaries

Derived daily rollups used for:

* trend analysis
* coaching continuity
* dashboard displays

Daily summaries are regeneratable.

⸻

import_batches

Tracks bulk imports.

Examples:

* historical ChatGPT conversations
* image-based weight logs
* JSON imports
* administrative uploads

GPT:

⸻

Historical Import

1. Export historical coaching chat.
2. Create import batch.
3. Parse messages.
4. Create log entries.
5. Extract fitness events.
6. Review low-confidence records.

⸻

Weight Log Import

1. Upload image.
2. GPT reads handwritten or printed weight data.
3. Create weigh-in events.
4. Associate records with import batch.
